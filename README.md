# InstaCrop

AI-powered ad creative engine. Upload one product image, get multiple banner and social ad creatives in every format — generated by Gemini.

---

## What it does

- Upload one product photo or ad image
- Choose output formats (Instagram, Facebook, banners, stories, etc.)
- Pick a generation mode (smart crop, background expand, recompose, product in scene)
- Generate multiple creative variants per format
- Preview, compare, and download all outputs as a ZIP

---

## Tech stack

- **Framework**: Next.js 15, React 19, TypeScript
- **Styling**: Tailwind CSS
- **Auth + DB + Storage**: Supabase
- **AI**: Google Gemini (image generation + analysis)
- **Image processing**: Sharp, Archiver

## Local setup

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/instacrop.git
cd instacrop
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env.local
```

Fill in `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
GEMINI_API_KEY=your_gemini_api_key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### 3. Set up Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** and run the contents of `supabase/schema.sql`
3. Go to **Storage** and create two buckets:
   - `uploads` — set to **private**
   - `outputs` — set to **public**
4. Enable **Google OAuth** in Authentication → Providers
5. Set the redirect URL to: `http://localhost:3000/auth/callback`

### 4. Get a Gemini API key

1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Create an API key
3. Add it to `.env.local` as `GEMINI_API_KEY`

### 5. Run the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Project structure

```
src/
├── app/                  # Next.js App Router pages + API routes
│   ├── page.tsx          # Landing page
│   ├── dashboard/        # Job history
│   ├── generate/         # New generation workspace
│   ├── results/[jobId]/  # Results viewer
│   ├── auth/             # Auth callback + error
│   └── api/              # REST endpoints (upload, jobs, generate, download)
├── components/
│   ├── ui/               # Button, Badge (base components)
│   ├── layout/           # Navbar
│   ├── upload/           # Dropzone
│   ├── generation/       # FormatSelector, GenerationSettings
│   └── results/          # OutputCard
├── lib/
│   ├── ai/               # Gemini provider, prompt builder, orchestrator
│   ├── db/               # Database queries (jobs, outputs, logs)
│   ├── storage/          # Upload/download helpers
│   ├── supabase/         # Supabase clients (browser + server)
│   ├── formats.ts        # Ad format presets
│   └── utils.ts          # cn(), formatBytes(), etc.
├── types/                # TypeScript types
└── middleware.ts         # Auth guard for protected routes

└── schema.sql            # Full database schema + RLS policies
```

## Architecture notes

- All AI logic lives in `src/lib/ai/` — never in components or pages
- The Gemini provider is isolated in `gemini-provider.ts` — swap it for another model by implementing the same interface
- Generation runs async; the UI polls `/api/jobs/[jobId]` every 3s until complete
- RLS policies in Supabase ensure users only access their own data
- Service role key is only used server-side (API routes) — never exposed to the browser
