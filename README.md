# InstaCrop

AI-powered ad creative engine. Upload one product image, get a full set of ad creatives in every format — generated by Gemini.

---

## What it does

1. Upload a product photo
2. Optionally draw around the product (lasso or rectangle) to isolate it
3. Choose output formats: Instagram, Facebook, banners, stories, etc.
4. Pick a generation mode (smart crop, background expand, recompose, product in scene)
5. Generate multiple creative variants per format — in parallel
6. Preview all outputs in the results gallery and download as a ZIP

## Local setup

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/instacrop.git
cd instacrop
npm install
```

### 2. Set up environment variables

```bash
cp .env.example .env.local
```

Open `.env.local` and fill in your values:

```env
NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
GEMINI_API_KEY=your_gemini_api_key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

See `.env.example` for all available variables and where to find each key.

### 3. Set up Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** and run the full contents of `supabase/schema.sql`
3. Go to **Storage** → create two buckets:
   - `Uploads` — set to **private**
   - `outputs` — set to **public**
4. Go to **Authentication → Providers** and enable **Google OAuth**
   - Set the redirect URL to: `http://localhost:3000/auth/callback`
5. Copy your project URL and keys into `.env.local`

### 4. Get a Gemini API key

1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Create a new API key
3. Add it to `.env.local` as `GEMINI_API_KEY`

### 5. Run the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Collaboration and branch workflow

This project is set up for two developers working in parallel:
- **Owner / backend dev** — AI pipeline, API routes, orchestrator, database
- **UI dev** — components, pages, styling, editor tools

### Branches

| Branch | Purpose |
|---|---|
| `main` | Stable, deployable code. Merge here when a feature is done. |
| `dev` | Active development branch. Both developers work off this. |
| `feature/...` | Individual feature branches, branched from `dev`. |

### First-time setup (owner)

```bash
git init
git add .
git commit -m "Initial InstaCrop rebuild"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/instacrop.git
git push -u origin main

# Create the dev branch
git checkout -b dev
git push -u origin dev
```

### UI developer onboarding

```bash
git clone https://github.com/YOUR_USERNAME/instacrop.git
cd instacrop
git checkout dev
npm install
cp .env.example .env.local
# Fill in .env.local with credentials shared by owner
npm run dev
```

### Daily workflow

```bash
# Pull latest before starting work
git pull origin dev

# Create a feature branch
git checkout -b feature/ui-results-redesign

# Work, commit often
git add .
git commit -m "feat: redesign output card layout"

# Push feature branch
git push -u origin feature/ui-results-redesign

# When done: open a PR from feature branch → dev on GitHub
```

### Backup (quick push to current branch)

```bash
git add .
git commit -m "wip: describe what you changed"
git push
```

## Architecture notes

- **No AI logic in components.** All Gemini calls live in `lib/ai/`. Components call API routes, which call the AI layer.
- **Async generation.** When the user hits Generate, the API responds immediately and the UI polls every 3 seconds.
- **Service role key is server-only.** It is never passed to the browser. Only API routes use it.
- **RLS policies** in Supabase ensure users can only access their own jobs and outputs.
- **Signed URLs** are used for private `uploads` bucket files. The `outputs` bucket is public.

## Deployment (Vercel)

1. Push `main` to GitHub
2. Import the repo at [vercel.com](https://vercel.com)
3. Add all variables from `.env.example` in Vercel's environment settings
4. In Supabase Auth settings, add your Vercel domain as an allowed redirect URL
5. Deploy
